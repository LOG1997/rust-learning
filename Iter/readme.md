# Rust中的迭代器Iter

## 核心概念

**迭代器**是rust中一种遍历集合（Vec、HashMap、数组等）或序列的工具，允许逐个访问集合中的元素，无需关心
集合的底层实现。

**核心特点**：

1. 惰性求值：迭代器本身不会主动遍历元素，只有调用消费型方法（`next()`、`collect()`、`for_each()`）才会执行遍历。
2. 零成本抽象：rust编译器会将迭代器优化成与手写循环一样的性能，不会有额外开销。

实现了`Iterator`特征才能称之为迭代器；实现了`IntoIterator`特征则可以通过`into_iter`，`iter`等方法变成一个迭代器。

## 基本使用

几乎所有集合类型都提供了获取迭代器的方法：
`iter()`：获取不可变引用的迭代器（遍历 &T）；
`iter_mut()`：获取可变引用的迭代器（遍历 &mut T）；
`into_iter()`：获取消费集合的迭代器（遍历 T，集合本身会被消耗）；

### 迭代器适配器

消费掉迭代器，然后返回一个新的迭代器

1. map()：对每个元素执行映射操作
2. filter()：过滤出满足条件的元素
3. take(n)：取出前n个元素
4. skip(n)：跳过前n个元素
5. chain()：拼接两个迭代器
6. enumerate()：获取元素的索引和值

### 消费者适配器

只要迭代器上的方法内部调用了next()方法，那么这就是消费者适配器，消费者适配器也会消耗掉迭代器上的元素。

1. collect()：将迭代器转为集合（如Vec、HashMap）
2. for_each()：对每个元素执行操作
3. sum()：求和
4. product()：求乘积
5. count()：统计元素个数
